# Readmail 2.1 (AI)

Система автоматической обработки писем-возвратов автозапчастей (оптовая торговля).
Читает входящую почту → **ИИ понимает** что за обращение → извлекает структурированные данные →
отдаёт чистый JSON в 1С. Форк readmail_v2, **только-ИИ** (паттерны-классификаторы убраны):
бизнес-решение по каждому письму принимает модель, а не regex.

> **Идея для оператора:** запустил автопилот — система сама 24/7 читает почту, обрабатывает,
> распределяет и выгружает в 1С, шлёт отчёт в Telegram. Человек заходит **только в «Ручной разбор»** —
> туда попадает лишь то, что ИИ не смог решить уверенно.

---

## Что → как → зачем

**Вход:** письма по IMAP — текст, HTML-таблицы, вложения (фото, PDF, Excel-акты), ссылки на
порталы возврата и фото-доказательства.

**Выход:** чистый документ возврата для 1С — **1 запчасть = 1 строка** (мультипозиционные письма
дробятся на отдельные заявки с привязкой к одной почте):
```json
{
  "buyer":   {"code": "...", "name": "..."},
  "claim":   {"kind": "quality_refusal", "kind_label": "Отказ клиента"},
  "document":{"number": "83904", "date": "15.06.2026", "type": "УПД"},
  "items":   [{"part_number":"VWAB035","brand":"FEBEST","product_name":"Сайлентблок","quantity":2}],
  "comment": "...", "source": {...}
}
```
> В UI причина возврата показывается **по-русски** (Брак/Маркировка/…), а в JSON для 1С —
> технический код (`quality_refusal`). Колонка «Выгружено» отмечает, что строка **реально** доставлена.

## Конвейер (AI-only)

```
IMAP-почта
  ① ПРИЁМ          imap_importer.py — raw_emails + вложения на диск (дедуп по message_id).
                   До ИИ бизнес-статус НЕ ставится: всё в «📥 Поступило в обработку».
  ② ИИ             ai_client.py — модель читает КАЖДОЕ письмо: тип (event_type),
                   причину (claim_kind), поля (документ/дата/артикул/бренд/кол-во), позиции.
                   ИИ — авторитет: перетирает черновик статики (static_hint).
  ③ ДОКАЗАТЕЛЬСТВА  дёшево без скачивания — сигналы по именам файлов/путям ссылок
                   (брак/акт/фото). Дочитка xls/акта в текст. Условный заход по ссылкам
                   (свежие + данные за ссылкой). Брак → vision по фото/докам.
  ④ РАСПРЕДЕЛЕНИЕ   folder_accounting.py — операторская ВОРОНКА (одна папка на письмо,
                   бизнес-смысл): Поступило → Проверяется AI → Ручной → Готово 1С → …
  ⑤ ЭКСПОРТ        чистый JSON → outbox → доставка файлом в 1С (status=sent).
```

**Воронка обработки** (вкладка 🗂 Воронка) — где письмо в бизнес-процессе:
Всего → Ожидают → Проверяется AI → Ручной разбор → Брак: проверить документы →
Возможная будущая проблема → Связки/диалоги → Готово к 1С → Служебные → Архив → Ошибки.
Клик по этапу → письма с пояснением «почему сюда / что делать».

**Типы:** new_return (брак/некондиция/пересорт/недовоз/отказ…), pre_delivery_refusal (отказ до
поставки), marking_request (ЧЗ/маркировка), correction_request (УКД/КСФ), ready_to_ship,
followup_dialog (диалоги), problem_notice (возможная проблема — не запрос на возврат),
supplier_report (прайсы/мусор).

## Модели и стоимость (routerai)

| Роль | Модель | Цена (вх/исх ₽ за 1М) |
|---|---|---|
| Текст | `qwen/qwen3-next-80b-a3b-instruct` | 9 / 73 |
| Визуал (фото/сканы брака) | `qwen/qwen3-vl-30b-a3b-instruct` | 12 / 48 |

Терминология RouterAI: **Входящие** = prompt (наш запрос), **Исходящие** = completion (ответ модели).
Полный прогон ~375 писем ≈ **27 ₽** (~7 коп/письмо). Цены редактируются в настройке
`AI_PRICE_RULES_JSON` — `pricing.py` читает её как единый источник; стата сверяется с RouterAI 1-в-1.
Ускорения vision: гейт (не читать фото, если поля уже полные) + сжатие 1280px + MoE-модель.

## Запуск

```bash
docker compose up -d --build          # контейнер readmail_21, порт 8766
```
- **Автопилот** (кнопка в UI) — 24/7: импорт → ИИ → распределение → 1С → отчёт в Telegram.
  При включённом автопилоте ручные кнопки конвейера блокируются.
- **Ручной прогон** (без багов вложенных соединений): `_classify_new.py` —
  фаза 1 скелет → фаза 2 ИИ (отдельным соединением) → очередь → выгрузка в 1С:
  ```bash
  docker exec -e PYTHONPATH=/app readmail_21 python3 /app/app/_classify_new.py 2000
  ```
- **Бесплатная пере-раскладка** из сохранённых ответов ИИ (0 вызовов модели):
  `_reapply_overlay.py` — после правок логики переприменяет overlay ко всем кейсам.

Данные (`./data`: БД, вложения, выгрузка 1С) — на host-bind, **не в git** (см. `.gitignore`).

## Ключевые модули

| Файл | Назначение |
|---|---|
| `ai_client.py` | промт + вызов модели, vision, токен-учёт |
| `classifier.py` | скелет (до ИИ — без бизнес-claim), `apply_ai_overlay` (ИИ перетирает) |
| `folder_accounting.py` | операторская воронка/папки (единый источник счётчиков) |
| `link_fetcher.py` | заход по ссылкам, парсер avtoto-рекламаций (артикул/бренд/док/фото) |
| `evidence_signals.py` | тип по именам файлов/ссылок без скачивания |
| `sender_profiles.py` | лёгкие пресеты под `@домен` отправителя |
| `pricing.py` | цены моделей (источник — `AI_PRICE_RULES_JSON`) |
| `db.py` / `main.py` | БД, FastAPI, очередь 1С, эндпоинты |

Промт-логика и спорные кейсы: `docs/issues_review_batch.md`, `docs/pipeline_logic.md`.
