# Идеи / на подумать (НЕ включено)

## Vision на приложенные изображения для любого возврата — ОТЛОЖЕНО (подумать)
Статус: **не включаем пока** (решение владельца).

Пробел (виден по V2): приложенные КАРТИНКИ читает vision только для defect-кейсов.
- фото-ССЫЛКИ рекламации (портал) → ИИ помечает mentions_photo — уже работает;
- фото-ФАЙЛЫ (брак.jpg, брак2.jpg) и СКРИНШОТЫ (Capture.PNG) вне defect-потока НЕ читаются.

Если в картинке список позиций / документ / данные — упускаем (текста мало).

Идея (когда решим включить): если у письма есть .png/.jpg и полей мало (или это
возврат с >0 изображений) → запускать ИИ-vision по картинке, доставать позиции/
доказательства, добивать поля. Закроет Capture.PNG-списки и брак-фото.

Риски: стоимость vision, латентность, ложные срабатывания на лого/подписях
(пистоп.png — лого, не документ). Нужен фильтр «значимая картинка vs лого».

Примеры из V2: id80771 (Capture.PNG + «ЧЗ не передан»), id81150 (брак.jpg ×3).
