# Логика обработки письма (readmail 2.1 AI-only)

Зафиксировано перед коррекцией. Версия 2.1 — без паттерн-движка; все поля
извлекает ИИ. Принцип экономики: **запрос дешевле ответа** → во вход кладём всю
конкретику (правила, примеры, список вложений, ссылки), на выходе — короткий JSON.

## 1. Первый прогон (один богатый запрос к ИИ)

В payload первого прогона уходит:
- `subject`, `from`, `text` (видимый верхний текст, без цитат);
- **`attachments[]`** — `{filename, content_type, size}` для каждого вложения;
- **`links[]`** — ссылки из тела (возвратные порталы, фото).

Промт обязан:
1. **Извлечь номера из ТЕМЫ и тела:**
   - «Претензия №N» / «Рекламация №N» → `claim_number` (это strong_key);
   - «по документу №N», «поступление №N» → `document_number` (номер поставщика, приоритетнее внутреннего УПД);
   - «Запрос на возврат N», «Заявка №N» → `client_request_number`;
   - артикул в теме («Арт. X», «при приемке: X») → `part_number`.
2. **По именам вложений** сразу проставить типы документов
   (`заказ-наряд установки/снятия`, `акт/заключение сервиса`) и `defect_documents_status`,
   если файлы названы осмысленно — тогда vision на первом прогоне не нужен.
3. **По ссылкам** выставить `mentions_return_link`.
4. **Классификация-исключения:**
   - «ТОВАР ПРИНЯТ С ДЕФЕКТАМИ … НЕ является запросом на возврат / не требует ответа»
     (trinity-parts) → `event_type=info_only`, `requires_action=false`
     (накладную/артикул всё равно извлечь как контекст).

## 2. Развилка после первого прогона

- **A → `needs_review`** только если данных реально нет (а не из-за слепого промта).
- **B → брак:** `defect_documents_flag` даёт состояние:
  `absent | present_unverified | partial | complete`.

## 3. Пробивка вложений (если по именам тип не ясен)

Порядок чтения (настройки `defect_read_pdf_first`, `defect_read_images_order=first_last_then_inner`,
лимит `max_defect_images_per_case`):

1. Есть явный PDF/документ → читаем его первым.
2. Нет явного PDF → берём **первый и последний** вложения.
3. Первый оказался документом → второй вероятно тоже документ → читаем дальше по порядку.
4. Первый — просто фото → **прыгаем в конец**.
5. Конец — документ → смотрим **предпоследний**.
6. Лимит изображений на кейс — `max_defect_images_per_case`.

Текст из PDF гнать **через ИИ-промт** (в AI-only `extract_fields` отключён → нельзя
использовать его для разбора PDF).

## Список правок к этой логике (что меняем)

1. `ai_client.py` — в payload первого прогона добавить `attachments[]` + `links[]`.
2. `ai_client.py` — в промт: извлечение номеров из темы + пример auto-sputnik + правило trinity «info_only».
3. `imap_importer.py` — реализовать порядок `read_pdf_first` + `first_last_then_inner` в `_run_vision_on_attachments` (сейчас плоский цикл).
4. `imap_importer.py` — ветку PDF-текста гнать через ИИ-промт, а не через мёртвый `extract_fields`.
5. `main.py` — батч вернуть на **полный** промт (откат компактного: вход дешевле выхода).
