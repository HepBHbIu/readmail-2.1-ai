# Панель Readmail (TUI) — инструкция мини-админа

Визуальная панель управления прямо в терминале (без графической оболочки).
Сверху — живые цветные блоки состояния, снизу — чат: по `/` всплывает меню функций.

**Режим: МИНИ-АДМИН · read-only.** Панель ничего не ломает:
- данные и БД **не меняются** (исключение — пауза/возобновление воркеров);
- **1С и AI не вызываются** ни одной командой;
- реальная доставка outbox **не запускается**;
- опасные операции (`deliver/reset/cleanup/mass-import/wipe`) **отключены**;
- секреты (пароли/токены) **не печатаются**.

Вся механика наследуется от `scripts/readmailctl.py` — панель только «надевает картинку»
поверх существующего бэкенда, сам бэкенд не дублируется и не изменяется.

---

## Запуск

```bash
readmail                 # глобальная команда (из любой папки)
# или
cd ~/Documents/readmail_v2 && ./readmail
# или напрямую
python3 scripts/readmail_panel.py
```

На сервере по SSH работает так же — это обычный текстовый терминал, графика не нужна.

### Установка глобальной команды на новой машине
```bash
cd <путь>/readmail_v2
ln -sf "$(pwd)/readmail" "$(dirname "$(which python3)")/readmail"   # в PATH текущего python
# системно (для всех пользователей, нужен sudo):
sudo ln -sf "$(pwd)/readmail" /usr/local/bin/readmail
```
Требуется пакет `textual` (`pip install textual`).

## Управление

| Клавиша | Действие |
|---|---|
| печатать текст | ввод в чат |
| `/` | открыть меню функций (фильтруется по мере набора) |
| `Enter` | выполнить команду / отправить |
| `↑ ↓` + `Enter` | выбрать пункт в меню |
| `Esc` | скрыть меню |
| `Ctrl+R` | обновить блоки (и так авто-обновление раз в 6 сек) |
| `Ctrl+L` | вернуть фокус в строку ввода |
| `Ctrl+Q` / `Ctrl+C` | выход |

Мышь и скролл чата работают.

## Блоки панели

- **СЕРВЕР** — статус, адрес host:port, LAN вкл/выкл, глобальная пауза.
- **ПОЧТА** — на сервере / в базе / дыра (недостающие письма) / карантин.
- **ОБРАБОТКА** — всего кейсов / без кейса / готово к 1С / в сверке.
- **OUTBOX / 1С** — в очереди / ошибки / отправлено / доставка ВКЛ/OFF.
- **AI** — включён, «паттерны = 0 токенов», вызовов и ₽ за сегодня.
- **ВОРКЕРЫ** — состояние import/stage2/ai/outbox (▶ работает / ■ стоп).

Лампочки в шапке: ● сервер · ● почта (зелёная если дыра = 0) · ● 1С (доставка).
Под чатом — строка с первым предупреждением (например «Автодоставка в 1С ВЫКЛЮЧЕНА…»).

## Команды чата (через `/`)

### Система
| Команда | Что делает |
|---|---|
| `/status`, `/refresh` | снимок системы / обновить блоки |
| `/doctor` | быстрый healthcheck |
| `/buckets` | визуальная бухгалтерия: где лежит каждое письмо |
| `/folders` | рабочие папки писем (сумма папок = total) |
| `/hidden [группа]` | «обработанные / не требуют действия» |
| `/pipeline [route] [reason]` | канонический конвейер (6 маршрутов) |
| `/decision case|raw <id>` | почему письмо/кейс попало в свой bucket |
| `/open` | URL веб-панели |
| `/help` | полный список команд |
| `/quit`, `/exit` | выход |

### Почта
| Команда | Что делает |
|---|---|
| `/mail` | состояние почты |
| `/reconcile` | снимок сверки IMAP↔БД (read-only) |
| `/backfill` | подсказка по dry-run backfill |
| `/quarantine` | карантин писем |

### Обработка
| Команда | Что делает |
|---|---|
| `/processing` | кейсы по стадиям |
| `/workers` | состояние воркеров |
| `/pause [worker]` | пауза (all\|import\|stage2\|ai\|outbox\|delivery\|telegram) |
| `/resume [worker]` | возобновить |
| `/worker-test` | read-only прогон стадий |

### Поиск и трассировка
| Команда | Что делает |
|---|---|
| `/search <запрос>` | единый поиск по письмам/кейсам/outbox/клиентам |
| `/trace case <id>` | цепочка письмо→кейс→outbox→попытки |
| `/trace raw <id>` | то же от письма |
| `/trace outbox <id>` | то же от outbox |

### Outbox / 1С (только просмотр)
| Команда | Что делает |
|---|---|
| `/outbox` | сводка очереди |
| `/outbox preview [N]` | предпросмотр payload (1С НЕ вызывается) |
| `/payload case|outbox <id> [--profile standard|minimal|debug]` | payload 1С (read-only) |
| `/delivery status` | режим интеграции / статус доставки |

### AI / настройки / логи
| Команда | Что делает |
|---|---|
| `/ai`, `/ai cost`, `/ai modes` | AI-мозг, стоимость, режимы |
| `/vision` | vision-статус |
| `/settings [раздел]` | настройки read-only, без секретов (runtime\|import\|workers\|ai\|onec\|auth\|paths\|env-safe) |
| `/logs`, `/logs errors` | последние события / ошибки |
| `/reports` | список отчётов |

### Отключено в режиме мини-админа
`/deliver` · `/reset` · `/cleanup` · `/mass-import` · `/ai batch` — вызывают опасные операции,
в панели заблокированы. Делать их — только через отдельный confirm-flow вне панели.

## Файлы

- `scripts/readmail_panel.py` — приложение панели (Textual).
- `readmail` — лаунчер (bash), он же глобальная команда через симлинк.
- `scripts/readmailctl.py` — механика (статус-aggregator + диспетчер команд), общая с CLI.
